# 百家姓起源图谱项目提示词

你是一个资深前端工程师和信息可视化设计师。请基于 Next.js + TypeScript + AntV G6，开发一个“百家姓起源图谱”开源项目。

## 项目目标

用图表/图谱的形式展示百家姓的起源关系。用户应该可以直观看到某个姓氏从哪里来、经过哪些源流、人物、地望、时代和起源类型形成，而不是只阅读大段文字。

## 技术栈

- Next.js App Router
- TypeScript
- Tailwind CSS
- AntV G6
- lucide-react

## 核心交互

默认不要把百家姓全部节点一次性铺开。首页只展示高层结构，包括：

- 古姓源头
- 姓氏源流，例如姬姓源流、姜姓源流、嬴姓源流、子姓源流、妫姓源流
- 起源类型，例如封地得姓、官职得姓、祖名得姓、赐姓、改姓避讳、民族融合
- 历史阶段，例如上古、先秦、秦汉、魏晋南北朝、隋唐

用户点击高层节点后，再局部展开代表姓氏。画面应保持克制、清晰，不要变成密密麻麻的大网。

## 搜索功能

必须支持按姓氏搜索。

搜索行为：

1. 输入汉字或拼音，例如 `马` / `ma`。
2. 展示匹配结果。
3. 选择某个姓氏后，图谱进入该姓氏的起源路径视图。
4. 右侧详情面板展示该姓氏的起源摘要、起源类型、时代、地望、人物和关系说明。

搜索结果不应该只是打开文字详情，而应该在图谱上呈现出清晰的来源路径。

## 选中姓氏后的图谱结构

选中某个姓氏后，不要继续显示凌乱的大网。应切换成整齐的分栏式路径图。

推荐结构：

```text
源流 / 类型 / 时代  ->  人物 / 地望 / 起源证据  ->  目标姓氏  ->  关系说明
```

以“马”姓为例，图谱应直观展示：

```text
嬴姓源流
封地得姓        ->  赵奢 / 赵国马服  ->  马  ->  赵 / 秦 / 梁
先秦

民族融合        ->  民族融合支流      ->  马
魏晋南北朝
```

马姓主要来源示例：

- 主要源流：嬴姓源流
- 起源类型：封地得姓
- 历史阶段：先秦
- 关键人物：赵奢
- 地望/封号：赵国马服
- 摘要：赵奢因功受封马服君，其后人有以封号或封邑中的“马”为氏者。
- 支流：历史上亦有复姓简化或族群融合过程中改为马姓的支流。

## 关系说明

不要在图谱边上只写“相关”。“相关”太泛，用户看不懂。

相关姓氏必须有明确关系标签和说明。例如马姓：

- 赵：赵氏近支  
  说明：马姓一支出自赵奢马服君叙事，与赵氏关系最直接。
- 秦：同属嬴姓源流  
  说明：秦姓一支与赵氏同归嬴姓古国支派。
- 梁：同源支脉  
  说明：梁姓一支也与嬴姓封国相关。

图谱边上的文字应使用这些具体标签，例如“赵氏近支”“同属嬴姓源流”“同源支脉”，而不是“相关”。

## 图谱布局要求

图谱必须稳定、整齐、可读。

避免使用持续迭代的力导向布局作为主展示方式，因为打开页面时节点会抖动，阅读体验很差。

推荐：

- 默认高层图谱使用固定坐标或稳定布局。
- 选中姓氏后使用分栏式路径图。
- 节点按层级排列，不要互相压叠。
- 边尽量横向流动，减少交叉。
- 只高亮当前姓氏的直接起源路径和证据节点。
- 非当前重点节点应弱化或隐藏，避免干扰。

## 节点类型建议

不同类型节点应有不同视觉语义：

- 姓氏节点：白底深色边框
- 姓氏源流节点：紫色
- 起源类型节点：青绿色
- 历史阶段节点：灰蓝色
- 起源证据节点：橙色
- 人物节点：蓝色
- 地望/封邑节点：绿色

## 数据结构要求

姓氏数据应结构化，方便后续补全百家姓。

示例：

```ts
{
  id: "ma",
  name: "马",
  pinyin: "ma",
  brief: "马姓常见源流归入嬴姓赵氏系统...",
  origins: [
    {
      kind: "place",
      sourceRoot: "嬴姓源流",
      period: "先秦",
      place: "赵国马服",
      ancestor: "赵奢",
      summary: "赵奢因功受封马服君，其后人有以封号或封邑中的“马”为氏者。"
    }
  ],
  relatedSurnames: [
    {
      name: "赵",
      label: "赵氏近支",
      note: "马姓一支出自赵奢马服君叙事，与赵氏关系最直接。"
    }
  ]
}
```

## 页面结构

页面建议分三栏：

- 左侧：项目标题、搜索框、高层结构列表、起源类型筛选
- 中间：G6 图谱画布
- 右侧：当前姓氏详情和关系说明

第一屏就是可用的图谱应用，不要做营销式 landing page。
